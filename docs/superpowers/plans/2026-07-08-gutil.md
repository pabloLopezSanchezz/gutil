# gUtil Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and package a safe, cross-platform Go CLI that prepares Git conflict merges and opens VS Code only when conflicts exist.

**Architecture:** A standard-library dispatcher calls focused command packages. The conflict workflow uses typed Git and editor interfaces backed by an injectable process runner, enabling deterministic unit tests and real temporary-repository integration tests.

**Tech Stack:** Go 1.24, Go standard library, Git CLI, GitHub Actions, POSIX shell, PowerShell.

- [ ] **Step 3: Implement the minimal dispatcher**

```go
const usage = `Usage: gutil <command> [options]

Available commands:
  conflict    Prepare or inspect a conflict merge
  version     Print the gUtil version
  help        Show this help
`

type ConflictCommand interface { Run([]string) int }

func Run(args []string, version string, stdout, stderr io.Writer, conflict ConflictCommand) int {
    if len(args) == 0 { fmt.Fprint(stderr, usage); return 2 }
    switch args[0] {
    case "help", "--help", "-h": fmt.Fprint(stdout, usage); return 0
    case "version": fmt.Fprintf(stdout, "gutil %s\n", version); return 0
    case "conflict":
        if conflict == nil { fmt.Fprintln(stderr, "Conflict command is unavailable."); return 1 }
        return conflict.Run(args[1:])
    default: fmt.Fprintf(stderr, "Unknown command %q.\n\n%s", args[0], usage); return 2
    }
}
```

Create a minimal `main` with `var version = "dev"` and `os.Exit(cli.Run(...))`.
